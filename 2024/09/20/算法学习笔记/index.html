排序算法

生成随机数组用于测试

选择排序

算法步骤：

  1. 首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置。
  2. 再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。
  3. 重复第二步，直到所有元素均排序完毕。

时间复杂度：O(n²)

空间复杂度：O(1)

冒泡排序

算法步骤

  1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
  2. 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元素会是最大的数。
  3. 针对所有的元素重复以上的步骤，除了最后一个。
  4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

时间复杂度：O(n²)

空间复杂度：O(1)

  • 异或的应用：找出数组中出现奇数次的数

插入排序

算法步骤

  1. 将第一待排序序列第一个元素看做一个有序序列，把第二个元素到最后一个元素当成是未排序序列。
  2. 从头到尾依次扫描未排序序列，将扫描到的每个元素插入有序序列的适当位置。（如果待插入的元素与有序序列中的某个元素相等，则将待插入元素插入到相等元素的后面。）

可以看作指定范围内的冒泡排序

时间复杂度：O(n²)

空间复杂度：O(1)

归并排序

算法思想

分成两块区域，先让左侧排好序，再让右侧排好序，最后合并排序。

算法步骤

  1. 申请空间，使其大小为两个已经排序序列之和，该空间用来存放合并后的序列；
  2. 设定两个指针，最初位置分别为两个已经排序序列的起始位置；
  3. 比较两个指针所指向的元素，选择相对小的元素放入到合并空间，并移动指针到下一位置；
  4. 重复步骤 3 直到某一指针达到序列尾；
  5. 将另一序列剩下的所有元素直接复制到合并序列尾。

时间复杂度：O(nlogN)

空间复杂度：O(N)

  • 非递归方式实现

二分法查找

适用于在有序数组或列表中查找目标元素的位置。其核心思想是通过每次将搜索区间减半，从而显著减少查找的时间复杂度

算法步骤

  1. 定义左右两个指针 left 和 right，分别指向数组的起始位置和末尾位置
  2. 计算中间点：根据左右指针，计算中间位置 mid，一般使用公式 mid = left + (right - left) / 2，防止直接 (left + right) / 2 可能导致整型溢出
  3. 比较目标值与中间值
    • 若目标值等于 mid 位置的值，直接返回该位置；
    • 若目标值小于 mid 位置的值，说明目标值在左半部分，将右指针 right 移动到 mid - 1；
    • 若目标值大于 mid 位置的值，说明目标值在右半部分，将左指针 left 移动到 mid + 1。
  4. 结束条件：当 left 大于 right 时，表示未找到目标值，返回 -1 表示目标值不存在。

时间复杂度：O(log n)

空间复杂度：O(1)

扩展

  1. 在一个有序数组中，找某个数是否存在

  2. 在一个有序数组中，找>=某个数最左侧的位置

  3. 在一个有序数组中，找<=某个数最右侧的位置

  4. 局部最小值问题

    局部最小值：指该元素小于它相邻的两个元素（如果存在）。如果数组只有一个元素，或者第一个元素小于第二个元素，或者最后一个元素小于倒数第二个元素，这些情况会被特殊处理

递归时间复杂度估算

在估算递归算法的时间复杂度时，通常需要使用递推关系式和主定理（Master Theorem）来分析递归函数的执行次数。我们主要关注递归的深度和每个递归调用的时间开销。

以下是分析递归时间复杂度的常见步骤：

1. 递推关系式的构建：

首先，构建递推关系式来描述递归的执行方式。递推关系式的形式一般是 $$T(n)=aT\left(\frac{n}{b}\right) + O(n^d)$$

  • $T(n)$：表示处理规模为 ( n ) 的问题的时间。
  • a：表示递归调用的次数（即子问题的数量）。
  • b：表示每次递归调用中问题规模缩减的倍数（即子问题的规模相对于原问题的比例）。
  • $O(n^d)$：表示每次递归调用中额外的工作量，如分解、合并、处理当前问题所需的时间。

2. 使用主定理分析递推关系式：

递归的时间复杂度可以根据主定理划分为三种情况：

  • 情况 1：如果 $a > b^d$，则时间复杂度为 $O(n^{\log_b a})$。
  • 情况 2：如果 $a = b^d$，则时间复杂度为 $O(n^d \log n)$。
  • 情况 3：如果 $a < b^d$，则时间复杂度为 $O(n^d)$。

例子分析

例1：二分查找的递归复杂度

二分查找的时间复杂度递归关系式可以写为：$T(n) = T\left(\frac{n}{2}\right) + O(1)$

  • 每次递归将问题规模减半，所以 $b = 2$；
  • 每次递归只进行一次递归调用，故 $a = 1$；
  • 每次递归调用的额外操作为常数时间 $O(1)$，所以 $d = 0$。

根据主定理：

  • $a = 1$，$b^d = 2^0 = 1$，即 $a = b^d$，符合情况 2，所以时间复杂度为：$O(\log n)$

例2：归并排序的递归复杂度

归并排序的时间复杂度递归关系式为：$T(n) = 2T\left(\frac{n}{2}\right) + O(n)$

  • 每次递归调用会将问题分为两个子问题，故 $a = 2$；
  • 每次将问题规模减半，所以 $b = 2 $；
  • 合并两个子问题的时间是线性 $O(n) $，所以 $d = 1$。

根据主定理：

  • $a = 2$，$b^d = 2^1 = 2$，即 $a = b^d$，符合情况 2，所以时间复杂度为：$O(n \log n)$

例3：分治法求解乘法（Strassen’s Algorithm）

Strassen 矩阵乘法的递推关系式为：$T(n) = 7T\left(\frac{n}{2}\right) + O(n^2)$

  • 每次递归调用会将问题分为 7 个子问题，故 $a = 7$；
  • 每次将问题规模减半，所以 $b = 2$；
  • 额外操作的时间是 $O(n^2)$，所以 $d = 2$。

根据主定理：

  • $a = 7$，$b^d = 2^2 = 4$，即 $a > b^d$，符合情况 1，所以时间复杂度为：$$O(n^{\log_2 7}) \approx O(n^{2.81})$$

3. 递归时间复杂度分析的基本技巧：

  • 找到子问题的个数：即递归调用的次数 $a$。
  • 找到子问题的大小：问题规模每次缩小的倍数 $b$。
  • 计算每层递归的额外工作：每次递归调用之外的操作量 $O(n^d) $。
  • 应用主定理：根据 $a$、$b$、$d$ 的关系，使用主定理确定递归的时间复杂度。

通过这几个步骤，我们可以系统地估算递归的时间复杂度。
